# Grafana Dashboard Exports

This directory contains JSON exports of our project's Grafana dashboards. These files serve as both a "Source of Truth" for version control and as a backup mechanism.

## 📂 Directory Structure

| File/Pattern | Source | Purpose |
| :--- | :--- | :--- |
| `dashboards/manual-exports/` | Grafana UI (Manual) | Formatted, cleaned JSONs intended for sharing and manual imports. |
| `dashboards/automated-backups/` | API / CI Pipeline | Raw exports generated by the backup script for disaster recovery. |

---

## 🛠 Usage & Integration

### 1. Manual Exports (UI-based)
These files are located in `dashboards/manual-exports`. Use these if you want to mirror a dashboard to a different environment or share it with a teammate.
* **How to import:** 1. Open Grafana.
    2. Navigate to **Dashboards** -> **Import**.
    3. Upload the `.json` file from this folder.
* **Note:** These files have been stripped of specific `uid` and `datasource` IDs where possible to ensure they are portable across different Grafana instances.

### 2. Automated Backups (API-based)
These files are located in `dashboards/automated-backups/` and are exported by the export-dashboard.sh script. At the moment there is no automated extraction. I extract them as needed. 
* **Frequency:** On demand
* **Purpose:** These are raw snapshots used for version history and disaster recovery.

---

## 🚀 How to Export a New Version

If you have made significant changes to a dashboard in the UI and want to update the **Manual Export**:
1. Open the Dashboard in Grafana.
2. Click the **Share** icon in the top bar.
3. Select the **Export** tab.
4. Toggle **Export for sharing externally** to ON (this helps remove instance-specific data).
5. Save the file to `manual-exports/` and commit the change.
